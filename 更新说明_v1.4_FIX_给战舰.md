# AI-Pulse v1.4 修复更新说明

> 更新日期：2026-05-27  
> 版本：v1.4-fix  
> 打包文件：`ai-pulse-v1.4-fix-deploy.tar.gz`

---

## 更新背景

本次更新主要解决生产环境中发现的几个核心问题：

1. **定时更新失效**：用户在前端设置了8:00自动更新，但实际没有执行
2. **AI HOT内容质量**：65.9%的内容来自Twitter/X，这些推文内容本身较短（100-300字符），影响内容质量
3. **中止功能不完善**：点击中止只中止了当前信息源的更新，而不是整个更新任务
4. **内容质量待提升**：48.4%的内容长度<200字符，大量短推文和空内容影响用户体验

---

## 问题详情与修复方案

### 问题1：定时更新失效

**问题描述**：
- 用户在前端更新设置中配置了每天8:00自动更新+生成日报
- 但实际到8:00时没有执行任何更新操作

**根本原因**：
- 数据库中 `scheduled_enabled=false`（定时更新开关被禁用）
- 虽然 `interval_enabled=true`（间隔更新启用，每30分钟执行一次），但这与用户的定时更新需求不符
- 代码逻辑正确（从数据库动态读取 `scheduled_time`），但开关关闭导致功能不执行

**修复方案**：
- ✅ 启用定时更新：`scheduled_enabled=true`
- ✅ 禁用间隔更新：`interval_enabled=false`（避免与定时更新冲突）
- ✅ 保留间隔配置：`sync_interval=30`（方便用户切换回间隔模式）
- ✅ 确认自动日报：`auto_generate_report=true`

**验证方法**：
- 等待下一个8:00，检查是否自动执行更新
- 查看日志：`tail -f api_server.log | grep "定时任务"`
- 应该看到：`[定时任务] 开始执行定时更新（每天08:00）...`

---

### 问题2：AI HOT内容质量优化

**问题描述**：
- AI HOT数据源中65.9%（216条）是Twitter/X链接
- 这些推文本身内容就短（100-300字符），无法抓取完整正文
- 与正常URL内容（4000-8000字符）相比，质量差异明显

**根本原因**：
- Twitter/X需要登录才能抓取完整内容，当前爬虫无法抓取
- AI HOT提供的摘要本身就是推文内容，长度有限
- 所有推文按原分数入库，没有区分质量

**修复方案**：

1. **降低Twitter内容评分**
   - 对Twitter内容降低评分20%（`final_score * 0.8`）
   - 让高质量博客/新闻内容排名更靠前

2. **过滤低质量推文**
   - 只入库分数>=60的推文
   - 自动过滤低分、低质量的推文内容

3. **添加内容标记**
   - 为Twitter内容添加标签：`["社交媒体", "Twitter/X"]`
   - 在内容末尾添加提示：`📱 *社交媒体内容，建议结合原文URL查看*`
   - 方便前端筛选和用户识别

4. **保持摘要后备策略**
   - Twitter内容继续使用AI HOT摘要作为内容
   - 这是预期行为，不是bug

**效果预期**：
- Twitter内容占比会下降（低分推文被过滤）
- 用户在前端可以通过标签筛选"社交媒体"内容
- 高质量博客/新闻内容排名提升

### 问题3：去重功能致命错误（代码bug修复）

**问题描述**：
- `dedup.py` 中调用了不存在的 `storage.query_materials()` 方法
- 导致去重服务初始化失败，URL去重功能完全失效

**根本原因**：
- `HotStorage` 类没有 `query_materials()` 方法
- 这是代码迁移时的遗留问题

**修复方案**：
- 改为直接查询 `selected_items` 表
- 修复 `load_existing_urls()` 和 `load_existing_vectors()` 方法

**验证方法**：
- 同步时观察日志，应该看到：
  ```
  去重服务初始化完成: 已加载606条URL，0条内容向量
  ```

---

### 问题4：评分变量作用域错误（代码bug修复）

**问题描述**：
- `sync_aihot.py` 中 `final_score` 在语义去重检查时使用，但还未定义
- 导致同步流程抛出 `UnboundLocalError` 异常

**根本原因**：
- 代码顺序错误：先进行语义去重（需要 `final_score`），后计算评分

**修复方案**：
- 调整代码顺序：先计算评分，再进行语义去重检查
- 确保所有使用 `final_score` 的地方都已正确定义

**验证方法**：
- 运行同步流程，不应再出现 `UnboundLocalError` 异常

---

### 问题5：无正文检测逻辑不完善（代码优化）

**问题描述**：
- `backfill_content.py` 仅检查内容长度，未识别HTML链接-only的内容
- RSS源常见问题：内容包含 `<a href` 链接但没有实质文本

**修复方案**：
- 增加对 `<a href` 标签的检测
- 条件：`content LIKE '%<a href%' AND length(content) < 500`
- 准确识别链接-only内容，进行回填

**验证方法**：
- 运行 `python3 src/backfill_content.py --dry-run`
- 应该能正确识别所有无正文/链接-only内容

---

### 问题6：中止功能不完善（代码优化）

**问题描述**：
- 用户点击"中止"按钮后，只中止了当前正在处理的信息源
- 其他信息源继续抓取，整个更新任务没有立即停止

**根本原因**：
- RSS/Web并发抓取阶段：虽然检查了 `_fetch_cancel_event`，但只是跳出循环，**没有关闭线程池**
- 主线程等待阶段：使用 `t.join()` 会等待所有线程完成，没有检查中止状态
- 并发任务继续运行，导致中止效果不理想

**修复方案**：

1. **RSS并发抓取阶段**
   - 检测到中止信号时，取消所有未完成的Future：`f.cancel()`
   - 关闭线程池并取消待执行任务：`executor.shutdown(wait=False, cancel_futures=True)`

2. **Web并发抓取阶段**
   - 同上，中止时立即取消所有待执行任务

3. **主线程等待阶段**
   - 改为带超时检查的等待：`t.join(timeout=1.0)`
   - 每1秒检查一次中止状态
   - 检测到中止信号后立即退出，不再等待线程完成

**效果预期**：
- 点击中止后，立即取消所有待执行的抓取任务
- 并发线程池快速关闭
- 整个更新任务在几秒内完全终止

---

### 问题7：内容质量待提升（新增优化）

**问题描述**：
- 数据库中48.4%（305条）内容长度<200字符，质量参差不齐
- Twitter/X内容占65.9%，大量短推文（100-300字符）影响用户体验
- 部分RSS源只抓取了摘要，没有完整正文
- 前端没有短内容提示，用户不知道内容质量

**优化方案**：

1. **过滤过短Twitter推文**
   - 同步时过滤<100字符的Twitter推文
   - 只保留有意义的推文内容
   - 减少垃圾内容入库

2. **增强Backfill回填逻辑**
   - 检测<200字符的非社交媒体内容
   - 自动尝试重新抓取完整正文
   - 排除社交媒体链接（无法抓取）

3. **优化RSS全文抓取**
   - 改进判断逻辑：content>500 或 summary>500才认为是完整内容
   - 对短摘要的RSS条目自动抓取原文全文
   - 确保RSS源提供完整正文

4. **前端短内容提示**
   - 内容<200字符时显示"⚠️ 内容较短，建议查看原文"
   - 黄色警告标记，醒目但不干扰
   - 提升用户体验

**效果预期**：
- Twitter短推文减少，入库内容质量提升
- RSS源正文完整度提高
- 用户能直观看到内容质量提示
- 整体内容质量明显改善

## 部署步骤

### 1. 停止当前服务

```bash
# 如果使用systemd
sudo systemctl stop ai-pulse

# 如果直接运行
ps aux | grep api_server
kill <PID>
```

### 2. 备份当前版本

```bash
cd /path/to/ai-pulse
cp -r . /path/to/backup/ai-pulse-v1.3-backup
```

### 3. 解压新版本

```bash
cd /path/to/ai-pulse
tar -xzf ai-pulse-v1.4-fix-deploy.tar.gz
```

或手动覆盖关键文件：

```bash
# 覆盖修复的Python文件
cp src/dedup.py /path/to/ai-pulse/src/
cp src/sync_aihot.py /path/to/ai-pulse/src/
cp src/backfill_content.py /path/to/ai-pulse/src/
```

### 4. 修复数据库配置（关键！）

```bash
cd /path/to/ai-pulse
python3 -c "
import sqlite3
conn = sqlite3.connect('data/hot_pulse.db')
cursor = conn.cursor()

# 启用定时更新
cursor.execute(\"UPDATE system_config SET value='true', updated_at=CURRENT_TIMESTAMP WHERE key='scheduled_enabled'\")
print('✅ scheduled_enabled = true')

# 禁用间隔更新
cursor.execute(\"UPDATE system_config SET value='false', updated_at=CURRENT_TIMESTAMP WHERE key='interval_enabled'\")
print('✅ interval_enabled = false')

conn.commit()
conn.close()
print('数据库配置修复完成')
"
```

### 5. 重启服务

```bash
# 使用systemd
sudo systemctl start ai-pulse
sudo systemctl status ai-pulse

# 或直接运行
cd /path/to/ai-pulse
nohup python3 api_server.py > api_server.log 2>&1 &
```

### 6. 验证部署

```bash
# 检查服务状态
curl http://localhost:8887/health

# 检查定时任务配置
curl http://localhost:8887/api/v1/config

# 检查精选内容
curl http://localhost:8887/api/v1/list?mode=selected&limit=5

# 手动触发一次更新测试
curl -X POST http://localhost:8887/api/v1/fetch
```
