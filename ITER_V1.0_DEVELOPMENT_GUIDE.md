# 知识炼金术Agent V1.0 开发手册
## 🎯 项目定位
独立的确定性知识处理流水线，专门把杂乱的信息加工成高价值的自媒体视频生产素材，和AI视频生成Agent、行为观察助手完全平级独立，后续可无缝对接视频生成Agent形成"知识采集→加工→视频生成"全链路。
---
## 🧪 V1.0版本目标
**核心目标：打通「URL抓取→去噪转Markdown→结构化处理→存储」全链路，做可用的素材预处理流水线，快速验证价值。**
- 优先级最高：网页爬虫模块，支持知乎/公众号/掘金等常用平台抓取，自动去噪转干净Markdown
- 核心逻辑：三维分类/价值评级/核心观点提炼，输出结构化结果
- 输出：结构化数据存入SQLite，支持命令行单篇/批量处理
**非核心功能延后：关联映射/接口/Obsidian集成等都放到V1.1及以后版本，先把最核心的爬取+处理链路跑通。**
---
## 📋 核心工作流（5步确定性流程）
每一步输入输出100%明确，无模糊环节：
### 步骤1：素材采集（V1.0核心新增）
- 输入：单个URL / 批量URL列表文件
- 处理逻辑：自动抓取网页正文，去除广告/导航/推荐等噪音，提取标题/发布时间/作者等元数据，转为干净的Markdown格式
- 适配平台：优先支持知乎/微信公众号/掘金/36氪等常用内容平台
### 步骤2：多维分类（三维坐标法）
- 处理逻辑：调用LLM按三个维度自动分类，每个维度的可选值固定，禁止自由输出
  - **时间维度**：可选值`即时热点`/`近期趋势`/`经典常青`（自动根据文章内的发布日期判断，无日期则默认`近期趋势`）
  - **内容维度**：可选值`方法论`/`认知`/`案例`/`数据`/`软件工具`/`资源`（严格按洋葱模型分类）
  - **场景维度**：可选值`脚本素材`/`竞品分析`/`避坑指南`/`金句库`（AI自动推测）
- 输出：
  ```json
  {
    "classification": {
      "time_dimension": "近期趋势",
      "content_dimension": "方法论",
      "scene_dimension": "脚本素材"
    }
  }
  ```
### 步骤3：价值评级（1-5星）
- 处理逻辑：站在自媒体编导的角度，根据素材对视频生产的价值打分：
  - 5星（核武）：颠覆认知、独家数据、必做选题 → 优先使用
  - 4星（弹药）：优质论据、精彩案例 → 写作时引用
  - 3星（参考）：普通资讯 → 备用参考
  - 2星（噪音）：过时、重复内容 → 归档
  - 1星（垃圾）：错误信息 → 删除
- 要求：必须附带打分理由，不少于30字
- 输出：
  ```json
  {
    "rating": {
      "score": 5,
      "reason": "该文章提出了全新的AI视频生产流程，属于颠覆认知的核心方法论，非常适合做深度科普视频选题"
    }
  }
  ```
### 步骤4：深度梳理（内容萃取）
- 处理逻辑：
  1. 提取3-5个核心观点（Bullet points）
  2. 必须回答："如果用来做视频，这个素材能用在哪个环节？"，给出明确的使用场景建议
- 输出：
  ```json
  {
    "extraction": {
      "core_points": [
        "纯代码渲染PPT作为默认方案可以提升50%生成速度",
        "三维分类法可以彻底解决知识库检索困难的问题",
        "确定性状态机架构可以保证AI系统输出100%稳定"
      ],
      "video_usage": "可以作为AI视频生产效率优化类视频的核心论据，放在方法论讲解环节"
    }
  }
  ```
### 步骤5：关联映射（知识图谱，V1.1版本实现，V1.0暂不开发）
V1.0不实现该功能，延后到V1.1迭代，届时再补充详细规范。
---
## 📌 强制规范
1. **输出格式100%严格匹配**：所有LLM返回结果必须是标准JSON格式，禁止输出任何额外的解释性文字，格式错误自动重试3次
2. **LLM提示词硬约束**：所有分类、评级的判定标准必须写死在提示词中，禁止LLM自由发挥，比如明确说明什么内容属于方法论、什么属于案例，5星的判定标准是什么
3. **技术栈统一**：和现有视频Agent技术栈完全一致，Python + DeepSeek API，复用现有项目的`llm_client.py`等基础工具类，减少重复开发
4. **预留对接字段**：输出JSON的字段和视频Agent的Writer Agent输入字段保持兼容，后续不需要二次转换即可直接对接
---
## ✅ 验收标准
1. 功能验收：
   - 命令行输入Markdown文本和3个笔记标题，自动走完所有流程，输出完整的JSON结果
   - 每一步输出格式100%符合要求，无格式错误
   - 连续运行5次，同一输入的分类、评级结果一致，无随机差异
2. 效果验收：
   - 🎯 爬虫准确率≥95%：抓取的正文无广告/噪音，元数据（标题/时间/作者）提取准确，Markdown格式干净
   - 分类准确率≥85%，和人工判断结果一致
   - 评级吻合度≥80%，和人工打分偏差不超过1星
   - 核心观点提取覆盖率≥90%，视频使用场景建议符合预期，有实际参考价值
3. 代码验收：
   - 所有提示词单独存放在`prompts/`目录下，不硬编码在逻辑中
   - 代码结构清晰，有完整注释，符合PEP8规范
   - 有测试脚本，可一键运行验证全流程
