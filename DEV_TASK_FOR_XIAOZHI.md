# 给小治的知识炼金术Agent V1.0开发指令
## 👋 任务说明
你负责知识炼金术Agent V1.0逻辑验证版的开发，严格按照《ITER_V1.0_DEVELOPMENT_GUIDE.md》开发手册和本指令执行，所有要求必须100%遵守。
---
## 📌 基础开发规范（必须严格执行）
1. **代码规范**：严格遵循Python PEP8规范，关键逻辑加中文注释，核心步骤写清楚输入输出说明，不写冗余代码
2. **仓库管理**：基于master分支切开发分支`dev/xiaozhi_v1.0`，每次提交写明确的提交信息，比如「feat: 实现三维分类逻辑」「fix: 修复星级判断不准的问题」，完成一个模块提交一次
3. **版本对齐**：所有依赖版本写在`requirements.txt`，写清楚最低版本要求，避免环境问题
4. **LLM Client复用**：直接复制AI视频项目的`llm_client.py`实现，保持所有系统的LLM调用逻辑统一，不需要重新开发
5. **配置规范**：用`config.yaml`统一管理所有配置：DeepSeek API Key、模型参数、提示词文件路径、输出格式要求，所有配置可修改，无硬编码
---
## 🎯 V1.0开发优先级（严格按顺序做，不做多余功能）
**V1.0核心目标：只做逻辑验证，跑通单篇素材的「分类→评级→核心提取」核心链路，不需要前端/不需要关联映射/不需要批量处理**
### 🔹 第一步：基础框架搭建
- 封装统一LLM Client，支持调用DeepSeek模型（和现有其他系统的LLM Client对齐，方便后续统一维护）
- 实现配置文件加载，模型参数、提示词等都写在yaml配置文件里，不要硬编码
### 🔹 第二步：核心逻辑开发（按5步工作流，先做前3步，第4步只做核心观点提取+适用场景标注，第5步关联映射V1.0暂时不做）
- 文本预处理模块：输入Markdown文本，自动去噪、提取正文
- 三维分类模块：输入文本，输出时间/内容/场景三个维度的标签，严格对齐开发手册的分类选项
- 价值评级模块：输入文本，输出1-5星评级+评级理由
- 核心提炼模块：输入文本，输出3-5条核心观点+视频适用场景说明
### 🔹 第三步：命令行入口开发
- 实现`python main.py process --file test.md`命令，处理单篇Markdown，打印每一步的处理结果
- 实现`python main.py batch --dir test_samples/`命令，批量处理目录下的文件，输出JSON格式的处理结果，方便批量验证准确率
### 🔹 第四步：接口开发
- 按开发手册的接口规范实现`/api/materials/query`查询接口，返回标准JSON格式
- 不需要用户管理、不需要权限、不需要复杂存储，用SQLite存处理好的素材即可
---
