## 📌 基础开发规范（必须严格执行）
1. **代码规范**：严格遵循Python PEP8规范，关键逻辑加中文注释，核心步骤写清楚输入输出说明，不写冗余代码
2. **仓库管理**：基于master分支切开发分支`dev/xiaozhi_v1.0`，每次提交写明确的提交信息，比如「feat: 实现三维分类逻辑」「fix: 修复星级判断不准的问题」，完成一个模块提交一次
3. **版本对齐**：所有依赖版本写在`requirements.txt`，写清楚最低版本要求，避免环境问题
4. **LLM Client复用**：直接复制AI视频项目的`llm_client.py`实现，保持所有系统的LLM调用逻辑统一，不需要重新开发
5. **配置规范**：用`config.yaml`统一管理所有配置：DeepSeek API Key、模型参数、提示词文件路径、输出格式要求，所有配置可修改，无硬编码

## 🎯 V1.0开发优先级（严格按顺序做，不做多余功能）
**V1.0核心目标：打通「URL抓取→处理→存储」全链路，先做最能快速验证价值的爬虫模块，不需要前端/不需要关联映射/不需要接口**
### 🔹 第一步：爬虫模块开发（最高优先级，先做这个）
- 实现网页抓取能力：输入任意URL，自动抓取正文，去除广告/导航/推荐/评论区等噪音
- 元数据提取：自动提取标题、发布时间、作者、来源平台
- 格式转换：抓取结果自动转为干净的Markdown格式
- 平台适配：优先适配知乎/微信公众号/掘金/36氪等常用内容平台
- 命令行支持：新增`crawl`命令，支持两种用法：
  - 单URL抓取：`python main.py crawl --url https://xxx.com`
  - 批量抓取：`python main.py crawl --batch urls.txt` 批量处理URL列表
### 🔹 第二步：基础框架+核心处理逻辑开发
- 封装统一LLM Client，支持调用DeepSeek模型（复用AI视频项目的已有实现，不需要重新开发）
- 实现配置文件加载，模型参数、提示词等都写在yaml配置文件里，不要硬编码
- 三维分类模块：输入Markdown文本，输出时间/内容/场景三个维度的标签，严格对齐开发手册的分类选项
- 价值评级模块：输入文本，输出1-5星评级+评级理由
- 核心提炼模块：输入文本，输出3-5条核心观点+视频适用场景说明
### 🔹 第三步：处理链路打通+存储
- 爬虫输出的Markdown自动传入核心处理模块，输出完整的结构化结果
- 结构化结果自动存入SQLite数据库，包含原始内容、元数据、分类/评级/核心观点等所有字段
- 命令行入口保留`process`命令，支持处理本地Markdown文件
### 🔹 第四步（V1.0不做，延后到V1.1）：接口开发/Obsidian集成等非核心功能
